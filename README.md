# PDFSplitter v3.1 — AI 智能 PDF 拆分工具

一个基于 Python + tkinter 的 Windows 桌面工具，利用**大语言模型（LLM）自动识别 PDF 扉页结构**，实现按章节/扉页智能拆分。支持手动指定页码和按数量平均拆分，适用于建筑图集、技术规范、报告合集等需要按篇章拆分的 PDF 文档。

---

## 核心功能

### 三种拆分模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **AI 自动识别扉页** | 调用 LLM 分析每页内容与布局，自动识别扉页位置并提取标题 | 任意 PDF，无需预先知道页码结构 |
| **手动指定页码** | 输入逗号分隔的页码（如 `1,15,30,40,47`），按指定位置切分 | 已知扉页页码 |
| **按数量平均拆分** | 输入拆分份数，自动计算每份起止页（余数分配到前几份） | 快速等分大型 PDF |

### AI 扉页识别（核心亮点）

- **双路径智能检测**：
  - **文字版 PDF**：提取每页文本 + 5 项布局指标（字符数、文字块数、文字面积比、图片数、居中状态），发送给 LLM 综合判断
  - **扫描件 PDF**：自动检测扫描件（>50% 页面文字 <10 字符），将每页渲染为图片以多模态方式发给视觉模型
- **支持多种 LLM 接口**：OpenAI / DeepSeek / 硅基流动 / 其他 OpenAI 兼容 API
- **扉页判定策略**：基于稀疏度、排版一致性、内容对比等方法，兼顾问卷的章节标识（第X章、Chapter X 等）
- **可配置参数**：模型、温度、最大 Token 等

### 辅助功能

- 统一保存路径设置（一键为所有拆分文件选择目录）
- 文件名修改时自动同步保存路径
- 路径冲突检测
- 进度条 + 状态栏实时反馈
- API Key 安全显示（掩码处理）
- 按钮悬停 / 按压视觉特效

## 技术架构

```
┌──────────────────────────────────────────────┐
│             tkinter 桌面 GUI                  │
│  文件选择 → API配置 → 拆分方案 → 一键拆分     │
├──────────────────────────────────────────────┤
│              业务逻辑层                        │
│  PDF读取/拆分 · LLM调用 · 拆分方案生成         │
├──────────────┬───────────────────────────────┤
│   PyMuPDF    │    urllib (HTTP Client)        │
│ (fitz)       │    → OpenAI 兼容 API           │
├──────────────┴───────────────────────────────┤
│              LLM 服务（远程）                   │
│   OpenAI / DeepSeek / 硅基流动 / 其他          │
└──────────────────────────────────────────────┘
```

**技术特点：**
- 单文件项目（1206 行），零额外 UI 框架依赖（仅 tkinter 标准库）
- 使用 `urllib.request` 而非 `requests`，减少打包依赖体积
- PyInstaller 打包为独立 `.exe`（31.3 MB，含 PyMuPDF），免安装直接运行

## 依赖环境

### 运行已打包的 .exe

无需安装任何依赖，直接双击运行。仅限 Windows。

### 从源码运行

| 依赖 | 用途 | 安装方式 |
|------|------|----------|
| Python 3.10+ | 运行环境 | [python.org](https://python.org) |
| PyMuPDF (fitz) | PDF 读取、拆分、页面渲染 | `pip install PyMuPDF` |
| tkinter | GUI 框架 | Python 标准库，无需额外安装 |

```bash
pip install PyMuPDF
python pdf_splitter.py
```

或使用项目自带的启动脚本：

```bat
run.bat
```

> **注意**：除 PyMuPDF 外无其他第三方依赖。HTTP 请求使用 Python 标准库 `urllib`，无需安装 `requests`。

## 快速上手

### 1. 配置 API

首次运行需配置 LLM API：

1. 点击 **「API设置」** 按钮
2. 填入 API 信息：
   - **API 地址**（Base URL），如 `https://api.openai.com/v1`
   - **API Key**
   - **模型名称**，如 `gpt-4o-mini`、`deepseek-chat` 等
3. 点击 **「测试连接」** 验证
4. 点击 **「保存」**

> 配置保存在程序同目录的 `config.json` 中。

### 2. 拆分 PDF

#### 方式一：AI 自动识别（推荐）

```
选择 PDF → 点击「AI 识别扉页」→ 等待分析完成 → 设置保存路径 → 点击「一键拆分」
```

- 程序自动检测 PDF 类型（文字版 / 扫描件），选择合适的分析路径
- 分析完成后，拆分表格自动填入扉页页码和标题
- 可在表格中手动修改页码和文件名

#### 方式二：手动指定页码

```
选择 PDF → 输入页码（如 1,15,30）→ 点击「生成拆分方案」→ 设置保存路径 → 点击「一键拆分」
```

#### 方式三：按数量平均拆分

```
选择 PDF → 输入份数 → 点击「平均拆分」→ 设置保存路径 → 点击「一键拆分」
```

### 3. 拆分结果

- 拆分后的 PDF 文件保存在指定的输出目录
- 文件名格式：`序号-标题.pdf`（AI 模式）或 `序号.pdf`（手动模式）
- 底部状态栏显示进度和完成状态

**打包配置要点：**

- 需确保 PyMuPDF 在当前 Python 环境中已正确安装，PyInstaller 会自动发现并打包

## 常见问题

**Q: AI 识别不出扉页？**

- 确认 API 配置正确（点击「测试连接」验证）
- 尝试降低 Temperature（如 0.0）提高确定性
- 对于扫描件 PDF，确保使用的模型支持视觉/多模态输入
- 扉页过于复杂或与正文差异不大时，可能无法识别

**Q: 拆分后文件打不开？**

- 检查保存路径是否有写入权限
- 确认 PDF 源文件未被其他程序占用
- 确保有足够的磁盘空间

**Q: 程序闪退或无法启动？**

- v3.1 打包版本可能缺少 PyMuPDF 运行时库，建议使用 v3.2
- 如从源码运行，确认 `pip install PyMuPDF` 已安装
